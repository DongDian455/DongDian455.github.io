OC与C语言的关系

OC是C语言的扩展，OC完全兼容C语言

  • OC支持C语言的所有变量类型
  • OC支持C语言的条件语句
  • OC支持C语言的关键字

常见关键字

OC中除了支持C语言的关键字之外，它还有自己特有的关键字，通常都以@开头

import

import是c语言中#include的加强版

  • include: 每次调用都会拷贝一份代码
  • import : 多次引用同一个文件只会拷贝一次

导入文件的方式：

双引号适用于本地文件（自己创建的文件）
<>适用指定的系统目录下查找对应的文件
@import 适用模块，能够提高速度和避免命名冲突

@class

@class与#import对比：@class只是让当前文件只是某个类名，具体类信息仍然不清楚。
使用@class指令会提高编译效率，因为不需要引入和处理整个类文件。

@autoreleasepool

Objective-C 中用于管理内存的一种机制。它创建一个自动释放池，确保在池的作用域内创建的对象在使用完后能够被及时释放，从而避免内存泄漏。

static

OC中的static不能修饰类属性和类的方法，但是可以在方法内外修饰变量,使得变量存储在常量池中。

  • 声明在方法内： 只能在方法类进行访问
  • 声明在方法外： 所有的方法都可以访问

    声明在一个类的方法内，那么同类名的访问都是同一个属性，如下：

selector

在 Objective-C 中，​​Selector（选择器）​​ 是一种特殊的类型（SEL），用于表示一个方法的名称（方法签名），类似于函数指针的概念。它不包含方法的实现，只代表方法的“名字”，用于在运行时动态调用方法。

语法

类的声明和定义

OC与C语言一样，需要区分声明和定义，声明通常放在头文件.h, 定义放在.m文件。

  • 对象声明使用@interface
  • 对象定义使用@implementation

alloc方法只是分配内存空间，init方法才是真正的调用类的构造函数进行初始化

类的方法

方法和类一样区分声明和定义。格式如下：
- (返回类型)方法名: (首个参数类型) 参数名1 形参名:(第二个参数类型)参数名2

OC中的形参数名比较奇怪，它的作用应该是用于描述两个参数之间的关系，而不像swift那样是给参数起一个别名

类方法与实例方法

类方法的定义为将普通方法开头的-修改为+

类的方法访问限制

  • oc的类方法，如果只定义了方法，没有方法的声明，那么方法就是私有的，只有类自身可以访问 (只在.m中实现,没有在.h声明）

类的变量

类的变量声明有两种

  • 传统的声明方式, 格式 :（类型）变量名
  • 使用@property自动生成， 格式: @property (类型)变量名

    注意：

    1. 使用@property 声明变量之后，编译器会自动生成以下划线开头的sette和getter方法, 现在都是推荐这种方法。
    2. 当外部访问类的成员的时候，通过.语法访问是不带下划线的，保持和.h中声明的一样，类内部访问则以下划线开头。
    3. 对象属性需要使用指针*声明

类变量的访问修饰符

在传统的变量声明方式中，可以给属性添加以下访问符号
@public,@private,@@protected,@@package

修饰符的作用域是从变量定义的开始，直接遇到下一个修饰符或者结束括号

而使用@peoperty关键字的情况下则不能使用上面的修饰符，只能通过别的方式限制:

  1. 只在.m文件中声明变量，则外部无法访问变量
  2. 在​​类扩展（Class Extension）中声明变量
  3. 使用​​readonly + readwrite​进行控制

@property参数

使用@property声明变量的时候，是可以携带参数，如：property(参数1，参数2，参数3) ，常见参数和使用方式：

  • atomic、nonatomic : 声明变量的原子性
  • assign、retain : 声明变量的内存管理方式，assign用于值类型，不持有对象不做引用计数。retain用于引用类型，持有对象​​，赋值时会自动 retain，释放时会自动 release。
  • strong、weak : 强弱指针
  • readwrite、readonly: 变量的读写权限控制

retain在 ​​ARC 下已被 strong取代​​，不推荐使用，而弱引用也不推荐声明为assign，而是用weak替代。
@property生成的默认方法是以_开头，而如果在.m文件再使用@synthesize定义这个变量，则可以不以_开头

类的构造方法

标准的构造方法定义模版有两种方法，分别为使用instanceType或者id。

注意在OC中，初始化可能会失败，所以必须判断父类是否初始化成功
instanceType:代表当前类型对象为当前类，可以作为返回值类型。
id: 任意类型，由于oc语言中，id类型的实例对象本质存放的是对象的类地址，所以可以用id定义任意类类型，并在程序运行的时候指向最终的对象。
