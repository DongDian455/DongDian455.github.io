开发环境

相关依赖

  • Ruby: 一种面向对象的脚本语言。MacOS默认已经支持运行Ruby，但是默认的版本往往过低，不支持使用Cocoapods，一般需要升级。
  • Homebrew : 它是MacOS下的一种软件包管理器，用于实现对软件包的安装、更新等管理；可以类比360软件管理器,使用它可以下载Gpg，Rbenv等
  • Rbenv : rbenv和rvm都是用来管理多个版本的 ruby 在用户目录的安装和使用，两者二选一使用。
  • RubyGems : 是 Ruby 依赖包管理工具。在 Ruby 中，包叫作 Gem，像Cocoapods、Fastlane等常用软件就是一种Gem. 可以通过gem install命令来安装
  • Bundler: 本身一种Gem, 是用于管理各个Gem的版本工具。
  • Cocoapods : iOS开发中最常用的第三方库管理工具，用于管理iOS项目工程所依赖的第三方库。它本身是由Ruby进行实现的，因此使用它需要先有Ruby环境。

在IOS工程中的使用方式：

  • 在IOS工程中，使用gem install bundle安装bundle
  • 执行bundle init，会生成一个GemFile文件，像 CocoaPods 和 fastlane 等依赖包，我们就可以添加到这个文件里面，如下：
  • 执行bundle install, 就会依照GemFile里的内容来安装各个 Gem以及其对应的版本。Bundler会自动生成一个Gemfile.lock文件来锁定所安装的Gem的版本。
  • 把Gemfile和Gemfile.lock一同提交到 Git，就可以保证其他开发人员都可以使用版本号一致的Gem。
  • 当使用各个Gem工具的时候，也需要使用 Bundler。例如在使用CocoaPods时要执行bundle exec pod，以保证我们使用的是项目级别而不是操作系统级别的Gem工具

配置启动图

使用Xcode创建新项目的时候，默认会自带LaunchScreen.storyboard文件，该文件为默认的启动页面，但是默认场景下是不会使用。
开启方式：

设置后默认的启动界面就是LaunchScreen.storyboard，然后在该storyboard添加对应的View来展示启动图即可

修改默认的ViewController

使用Xcode创建新项目的时候，默认会使用Main.storyboard为启动页之后的首个展示界面,如果需要修改，则需要在AppDelegate中替换为对应的ViewController

需要注意的是，Xcode13开始，创建新项目会使用到SceneDelegate,该功能是用于构建多窗口管理的，一般App开发很少使用，所以如果单纯开发App功能，可以把SceneDelegate相关的内容进行移除
