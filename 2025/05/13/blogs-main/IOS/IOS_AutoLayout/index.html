发表于2025-05-13|更新于2025-05-13|IOS
|阅读量:

translatesAutoresizingMaskIntoConstraints

该变量的作用是：是否将视图的自动调整大小的遮罩(autoresizing mask)转换为 Auto Layout 的约束，默认值为true

当该属性为 true 时，系统会自动通过视图的 autoresizing mask 创建一组视图的约束，这些约束是基于你提供的 frame、bounds、center 这些属性

需要注意的，当属性值为 true，指定 frame 且添加额外约束会出现约束冲突

所以，如果想使用 Auto Layout 动态计算、改变视图尺寸的话，必须将该属性值改为 false 。然后你只需提供无歧义、无冲突的约束即可。

抗拉伸/压缩

在AutoLayout的布局下，如果设置的约束无法确定ui视图的大小，那么就可以通过设置拉伸/压缩优先级来控制视图的大小

  • setContentHuggingPriority
    内容区域小于视图区域的时候使用（比如，TextView的内容宽度小于TextView的最大宽度）
    抗拉伸效果，优先级越高,视图就越会紧紧抱在一起，不会被拉伸

  • setContentCompressionResistancePriority
    内容区域大于视图区域的时候使用比如，TextView的内容宽度大于TextView的最大宽度）
    抗压缩效果，优先级越高，视图越不会被压缩，会尽可能展示全内容
